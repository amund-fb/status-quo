Notion 3.4: Workers functions, Calendar/Mail/Slack integration, faster MCP with admin controls

Notion released version 3.4 on April 14 with three meaningful changes for agent-based workflows. Workers are JavaScript/TypeScript functions that Notion AI agents can invoke to run real queries, calculations, and API calls — replacing approximated answers from visible text. New integrations connect Calendar, Mail, and Slack, allowing agents to read, summarise, and act across daily tools. MCP improvements add faster responses and new admin controls including auditing and approved-tool lists.

Workers functions make Notion AI a genuine computation layer. For client-facing dashboards, a Worker can now pull live data and run calculations inside Notion without a separate Python pipeline. Test with one client's weekly KPI summary — build the Worker, trigger it via MCP, and see if it replaces the current scripted export. The n8n MCP integration also enables connecting Notion agents to broader automation workflows without custom code.

⌘ Commands Worth Knowing

claude mcp add bigquery Register Google's official BigQuery MCP server in your Claude Code session. First step to querying client datasets directly from the terminal without a custom connector.
bq ls --project_id=PROJECT_ID List all datasets in a BigQuery project. Run this immediately after connecting the MCP server to confirm Claude can see the schema you want to expose.
claude mcp add slack Connect Slack's now-GA MCP server to your session. Enables real-time channel search, thread reads, and message drafting — all agent-native without OAuth setup.
claude --resume Resume your last Claude Code session with full context, open files, and tool state intact. Essential now that parallel sessions are live — don't recreate context you already built.
bq query --format=csv --max_rows=1000 'SQL' Export BigQuery results as CSV directly to the terminal. Pipe the output to Claude Code for on-the-fly analysis without opening a browser or a notebook.
gh run watch Stream a GitHub Actions run in real time from the terminal. Use this when a Claude Code session triggers CI — watch the pipeline without leaving the shell.
